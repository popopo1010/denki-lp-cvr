# LP作成リファレンス

施工管理キャリア系LP（電気工事士・施工管理・年収診断・Metaショート）を新規作成・改修するときの共通指針。  
エージェント向けの運用ルールは `CLAUDE.md`、本番URL・反映手順は `本番反映手順書.md` を参照。

---

## 1. このリポジトリのLP種別

| 種別 | パス例 | 用途 | 木下流での位置づけ |
|------|--------|------|-------------------|
| 通常LP（工種別） | `sekoukanri-kentiku/` など | SEO・汎用送客 | ロング寄り・専門性で信頼 |
| 通常LP（総合） | `sekoukanri/` | ハブ・広いキーワード | 選択肢が多く3秒判断は弱い→工種LPへ誘導 |
| 年収診断LP | `nenshu-shindan/...` | 損失回避・LINE導線 | インサイト＋結果→次アクション |
| MetaショートLP | `meta-lp/...` | Meta広告のリンク先 | 3秒・1メッセージ＋1CTA |
| 広告CR（録画用） | `ad-cr/` | 動画撮影のみ | **送客先にしない** |

再生成スクリプト:

- 工種別通常・年収診断: `python3 scripts/generate-sekoukanri-variants.py`
- Metaショート: `python3 scripts/generate-meta-lp.py`

## 2. 木下勝寿（北の達人）ナレッジ — LP設計に転用する部分

### 2.1 前提（何を借りて、何を借りないか）

- **借りる**: マーケの構造・改善の順序・心理トリガーの置き方
- **借りない**: 化粧品ECのコピーそのまま、社長ファン訴求、木下氏個人の実績数字
- **理由**: 北の達人はD2C購入、当社LPは**リード獲得（電話・LINE）**。ゴールが違う

参考:

- [木下勝寿 - Wikipedia](https://ja.wikipedia.org/wiki/%E6%9C%A8%E4%B8%8B%E5%8B%9D%E5%AF%BF)
- 書籍『ファンダメンタルズ×テクニカル マーケティング』（日経BP）

### 2.2 コアフレーム: ファンダメンタルズ × テクニカル

| 層 | 木下氏の定義 | 当LPでの具体 |
|----|-------------|-------------|
| **ファンダメンタルズ** | 誰に・何の悩み・何を約束するか | 工種・級・「無料」「転職しなくてもOK」をFVで一言化。広告文＝FV1行目 |
| **テクニカル** | CTR・離脱・CVRを数値で直す | ステップ別離脱、サンクス到達、LINE遷移をGTMで職種・`__LP_ID`別に計測 |
| **インサイト** | 本人も言語化していない欲求 | 「同じ資格でも年収200万差」「知らないまま損」など |
| **ペルソナ** | 同級生・上級生・未経験のモデル | 口コミを1級/2級・工種別に分け、読者が自分に重ねられるようにする |

**作業順序（木下流）**

1. ファンダメンタルズを決める（広告・FV・step01まで一貫）
2. テクニカルで計測し、ボトルネックのステップだけ直す
3. 装飾・アニメーションは最後（CVRに効くと証明できてから）

### 2.3 「3秒で離脱」ルール

スマホでは**最初の3秒で「自分向けか・無料か・次に何か」**が決まる。

| NG（情報過多） | OK（1画面） |
|----------------|------------|
| 信頼バー＋社会的証明＋通知＋バナー＋バッジ＋2問目の長い質問 | **1メッセージ**（例: 建築1級・2級の高待遇求人を無料紹介） |
| 11資格が並ぶstep01 | 工種LPで4〜6択＋「建築施工管理専門」バッジ |
| 広告「年収診断」→ 着地「お気持ちはどちら？」 | 広告コピーとFVの**完全一致** |

実装の対応:

- Meta送客 → `meta-lp/`（`meta-short-lp.css` で口コミ・FAQ等を非表示、コンパクトFV）
- 通常LPでもFV直下の信頼要素は**ステップ2以降**へ寄せる方針

### 2.4 心理トリガー（転職・求人LP向け）

| トリガー | 木下・北の達人での例 | 当LPでの使い方 |
|----------|---------------------|----------------|
| **損失回避** | 「知らないと損」 | 年収診断の `ns-fv__loss`、相場より安いかも |
| **温度差** | 購入意欲の段階 | step-first「転職したい / 情報収集」＋「転職しなくてもOK」 |
| **返報性** | 入力の見返り | 各ステップの `cvr-step-reward`（次に何が届くか） |
| **社会的証明** | 利用者数・満足度 | `cvr-social-proof`・口コミ（**数値は要確認**） |
| **専門性** | カテゴリNo.1感 | 工種別LP・許可番号・「○○施工管理専門」 |
| **具体レンジ** | 求人の給与幅明示 | 年収相場カード・「○○万〜」は根拠付きで |

### 2.5 木下氏の営業・採用LP出身から学ぶこと

- 求人情報誌営業 → **ユーザーの温度**（今すぐ転職 vs 情報収集）を最初に分ける設計は有効
- 採用LPの型（給与レンジ・未経験OK）→ **年収レンジ・1級/2級の違い**をFV〜step01で見せる
- 「社長直轄」訴求 → 当社は**第三者のキャリアサービス**なので、代わりに**厚労省許可・工種特化**で信頼

### 2.6 ロング vs ショートの使い分け

| チャネル | 推奨LP | 理由 |
|----------|--------|------|
| Meta / アプリ内ブラウザ | `meta-lp/` ショート | 3秒・スクロール少・1CTA |
| リマーケ・一部検索 | 通常LP（下部にFAQ・口コミ） | 検討時間はあるが冗長FVは離脱 |
| 年収診断キャンペーン | `nenshu-shindan/` | 損失フレーミング＋診断後LINE |

木下氏は条件次第でロングも有効と言うが、**広告クリック直後はショートが先に勝ちやすい**。

### 2.7 ファンダメンタルズチェックリスト（新規LP前）

- [ ] **誰向けか**が3秒でわかるか（工種・1級/2級・電気工事士など）
- [ ] **何が無料か**（求人紹介・年収診断・相談）が一言でわかるか
- [ ] **広告の1行目**とFVの1行目が同じか（UTM別出し分け含む）
- [ ] **転職しなくてもOK**が必要層に伝わるか（ハードル下げ）
- [ ] step01の選択肢は**スマホ1画面**に収まるか（目安4〜6個）
- [ ] 各ステップに**返報**（入力すると何が得られるか）があるか
- [ ] 口コミは**その工種・級**の声か（総合LPに全部載せない）
- [ ] 数値・許可・満足度は**根拠確認済み**か（`CLAUDE.md`）
- [ ] サンクス後の**次アクション1本**（LINE or 電話）が明確か
- [ ] `__LP_ID` が計測・Zapier用に一意か

### 2.8 テクニカル改善・A/Bの例

| テスト | 仮説 | 計測 |
|--------|------|------|
| FVコピー | 損失型 vs 獲得型 | step-first離脱率 |
| CTA文言 | 「無料でチェック」 vs 「30秒診断」 | step-first→step01 |
| LP長さ | 通常 vs meta-lp | フォーム完了率 |
| サンクス | 即LINE vs 説明後LINE | `line_redirect` |
| 工種LP | 総合 vs 建築専門 | `_lp` 別CVR |

### 2.9 当リポジトリへのマッピング（実装クラス・ID）

| 木下の意図 | HTML / CSS / JS |
|-----------|-----------------|
| 工種・専門性 | `cvr-specialty-badge`, `cvr-fv-note`, 工種別ディレクトリ |
| 返報明示 | `cvr-step-reward`（全ステップに統一） |
| 理由・不安解消 | `cvr-step-reason` |

## 3. 工種・級別LP（木下「ファンダメンタルズ」と候補者心理）

候補者は**フォームで11択できても**、「このサイトは自分の資格向けか」がFVでわからないと離脱しやすい。

| 課題 | 対策 |
|------|------|
| FVが「施工管理技士」総合 | 建築・土木・電気施工管理でURL分割 |
| 1級/2級の違いが見えない | 各工種LPのstep01で1級・2級を前面＋口コミも級別 |
| 電気工事士が施工管理LPに混在 | 施工管理LPには兼業向けとして残す場合あり。主訴求は `denkikouji/` |
| Metaで工種出稿 | `meta-lp/sekoukanri-{kentiku,doboku,denkisekou}/` |

**1級専用URLまで増やすか** → 広告費・検索ボリューム次第。多くの場合は**工種3本 × FVで1級寄せ文言**で足りる。

## 4. 口コミ・返報の書き方（木下: 同級生モデル）

### 口コミ（3タイプを工種LPごとに）

1. **同級生**: 「2級でも年収○万UP」（同じ級の読者）
2. **上級生**: 「同じ1級でも年収に○百万の差」（級内の上限意識）
3. **クロス**: 電気工事士のみ・兼業など工種LPの例外パターン

※ 金額・年齢・属性は**要確認**。ダミーを本番にしない。

### 返報（`cvr-step-reward`）の型

- step-first: この回答で → **転職時期に合った案内**（残りステップ数）
- step01: 次に届く → **その工種の1級・2級**に特化したリスト
- 中間: 回答後 → **絞り込み軸**（経験・エリア・年齢）
- 最終: 送信後 → **無料案内・営業電話なし**

年収診断は「診断結果」「適正年収レンジ」など**結果ワード**に寄せる。

## 6. 新規LP作成の最短手順

1. **ファンダメンタルズ**を1文で書く（例: 「建築2級向けに、転職しなくても無料で建築現場の求人相場を見せる」）
2. ベースHTMLを選ぶ（工種なら `sekoukanri-kentiku/index.html`）
3. `generate-sekoukanri-variants.py` または手動で `__LP_ID`・資格リスト・口コミ・返報を差し替え
4. Meta用なら `generate-meta-lp.py` で `meta-lp/` を生成
5. `node scripts/check-lp-guard.js` を通す
6. `本番反映手順書.md` にURL・GTMメモを追記
7. スマホで step-first → 送信 → サンクス・プライバシー導線を確認

## 7. やってはいけないこと（木下流＋当社ルール）

- 根拠のない数字をFVに載せる（木下の「98%はボツ」精神＝検証なき強い主張は避ける）
- `ad-cr/` を広告のリンク先にする
- 総合LP1本だけでMeta全工種広告を回す（広告とFVの不一致）
- 社長ストーリー・北の達人の実績をそのまま転用する
- ライブラリ追加や重いアニメで読み込みを遅くする
