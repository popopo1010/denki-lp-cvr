## 2. コアフレーム: ファンダメンタルズ × テクニカル

### 2.1 定義（木下氏）

| 層 | 定義 | やること |
|----|------|----------|
| **ファンダメンタルズ** | 商品・ペルソナ・インサイトから「誰に・何を・なぜ」を決める | 感情ベースのコミュニケーション設計（上流） |
| **テクニカル** | CTR・CVR・CPO等の数値から「どう伝えるか」を磨く | 1円単位で採算を見ながら運用（下流） |

**作業順序（木下流）**

1. ファンダメンタルズで仮説とコンセプトを固める（広告文＝FV1行目まで一貫）
2. テクニカルでボトルネックだけ直す（A/Bはレイヤーをずらさない）
3. 装飾・アニメは最後（CVRに効いたと証明してから）

### 2.2 当社（リード獲得LP）への翻訳

| 北の達人（D2C購入） | 電気工事バンク / 施工管理キャリア（リード） |
|---------------------|------------------------------------------|
| カゴ・購入ボタン | フォーム完了・サンクス予約・LINE登録 |
| LTV − CPO | 1リードあたり粗利 − CPA（電話・面談コスト込み） |
| 商品を買わない選択肢を潰す | 「転職しなくてもOK」「押し売りなし」「10分だけ」 |
| 上限CPOで広告停止 | 採算の合う広告だけ回す（運用ルール） |

**借りる / 借りない**

- 借りる: 構造・順序・心理トリガー・マイクロコピー・A/Bの考え方
- 借りない: 社長ファン訴求、北の達人の売上数字の転用、化粧品ECコピーのそのまま流用

## 3. ファンダメンタルズ手法（LP設計に直結）

### 3.1 ユーザーニーズの9段階分類

顧客の「悩みの温度」を9段階に分け、段階ごとに刺さるメッセージが変わる。

| 段階 | 状態（要約） | 転職LPでの例 |
|------|-------------|-------------|
| 1 | 課題に気づいていない | 広告では届かない |
| 2 | 気づくが一時的と思う | 「今の職場、本当にこのままでいい？」 |
| 3 | 自覚するが何もしていない | 「情報収集だけ」CTA |
| 4 | 対策を検討し始めた | フォーム step-first |
| 5 | かなり詳しい | 工種別LP・比較軸 |
| 6 | 手を打ち始めた（何か買った/登録） | 登録完了＝ここから |
| 7 | お気に入りがあり満足 | 現職維持でOK訴求 |
| 8 | 満足だが他に良いものは？ | 非公開求人・年収UP |
| 9 | 色々試したが満足なし | 体験談・再提案 |

**サンクス到達者はだいたい 6→8**。木下流では「登録直後は面倒・押し売り不安を潰す」が大分類の仕事。

出典: 書籍要約 [tjblog](https://tjblog.jp/2022/07/07/1580/)、[note要約](https://note.com/hideya11go/n/n09b00320c7ea)

### 3.2 4段階セールスコピー（競合を「個別商品」ではなく「選択肢グループ」で潰す）

| 段階 | やること | 薄毛例（木下・D2Cの会） | 当社サンクス例 |
|------|----------|------------------------|----------------|
| 大分類 | そもそもやらない理由を潰す | サロンは面倒・押し売り | 転職エージェントは押し売り・面倒 |
| 中分類 | 手段の比較 | サプリ vs 育毛剤 | 転職サイト alone vs 比較＋電話 |
| 小分類 | 製品タイプの比較 | サラサラ vs ベトベト液 | 電話必須 vs 10分任意 |
| 商品分類 | 自社商品を提示 | 当社の育毛剤 | 当社の非公開求人＋担当 |

**LPへの当てはめ**: 広告〜FVで大分類、フォームで中分類、サンクスで小分類＋商品（予約・LINE）。

出典: [Impress 後編](https://netshop.impress.co.jp/node/10721)

### 3.3 クリエイティブの階層（1次 / 1.5次 / 2次）

| 種類 | 情報量の目安 | いつ |
|------|-------------|------|
| フィールド情報 | 1000 | ユーザーインタビュー・競合・成分 |
| オリエン情報 | 100 | 社内に渡す要約 |
| 1次クリエイティブ | 10 | 誰に・何を・どう（初稿） |
| 1次リライト | 10 | 媒体違い・疲弊前の微修正 |
| 1.5次 | — | オリエンに戻って再構成 |
| **2次** | フィールドに戻る | ゼロから作り直し（木下氏はここを「マーケター」） |

**示唆**: サンクスCVRが止まったら装飾ではなく、2次＝「インサイトから順序を組み直す」。

### 3.4 小藪方式（啓蒙が必要な商品向け）

- 会話調・否定と肯定を交互に、「なぜこの非効率なやり方か」まで読ませる
- USPが直感で伝わるならストレートでよい
- **転職・施工管理**は啓蒙が要る領域 → サンクスの `<details>` 折りたたみや「比較しないと損」はこの系統（任意・短く）

### 3.5 コピーの順番

- LPフォームの定番フォーマットは**最後まで読まれない**
- **絶対伝えたいことは最初**（返報・無料・押し売りなし）
- 木下氏: エモーションリレー（感情の段階を順に運ぶ）

当社: `cvr-step-reward`、サンクスヒーロー3ステップ、予約前マイクロコピーはこの原則。

### 3.6 ペルソナ・USP（書籍・要約より）

- プロダクトのペルソナは **USP・ベネフィット起点で最大公約数**（細かすぎるとターゲットが狭まる）
- 細かいペルソナはメディア出稿用
- USPは4分類で整理（書籍）— 転職LPでは「工種特化・非公開・比較専門・無料」

## 4. テクニカル手法

### 4.1 A/Bテストの正しいやり方

| レイヤー | 例（フリース） | 当社例 |
|----------|---------------|--------|
| **何** | 軽さ vs 暖かさ | 損失型FV vs 獲得型FV |
| **どう** | 「200g」 vs 「スマホ1個分」 | 「10分予約」 vs 「無料相談」 |

**NG**: 「何」と「どう」を混ぜた比較（検証が無効になる）。

出典: [Impress 後編](https://netshop.impress.co.jp/node/10721)

### 4.2 利益の式（D2C原論の転用）

```
全体利益 = (LTV − CPO) × 獲得件数
```

- **売上最大化 ≠ 利益最大化**（CPOを上げると件数は増えるが1件利益が減る）
- **売上最小化、利益最大化**: 上限CPO超えの広告を止めると売上は半減しても利益は増える
- 当社: 「登録数だけ追う」より「予約完了・LINE・採算付きリード」をKPIに

### 4.3 テクニカル運用の3要素（要約）

1. **テクニカル運用**: 獲得単価・ROAS・件数の増減要因を理解
2. **KPI**: 利益を段階管理（5段階と書籍内）
3. **テクニカルクリエイティブ**: 着眼法・苦情法で改善

### 4.4 マイクロコピー（木下氏の代表事例）

| 事例 | 文言 | 効果 |
|------|------|------|
| 北の達人創業期 | 数量横に「お1人様2個まで」 | 売上 **1.5倍** |
| ボタン | 「注文する」→「試してみる」 | CVR **1.2倍** |
| 矢印付き | 「試してみる→」 | さらに **1.2倍**（合計約1.44倍） |

**原則**

- キャッチ・ボディではなく、**迷う直前**（ボタン横・プルダウン・チェックボックス）
- 「コスト」ではなく「知恵」
- 競合に真似されにくい（気づかれにくい）
- CVRを上げるなら**カゴ周り**から（D2C）→ 当社は**予約ボタン・LINEゲート・同意文**

出典: [microcopy.org](https://microcopy.org/thinking/mr_kinoshita/)

**当社サンクスへのマッピング（2026-06 実装）**

- ヒーロー: `押し売りなし` / `10分だけ` / `転職しなくてOK`
- CTA下: `合わなければその場でお断りください`
- 予約確定: `この日時で10分の電話を予約（無料）`
- カレンダー上: `日時を選ぶだけ（入力不要）`

## 5. 北の達人のLP（自社D2C）から読み取れる型

公式ECは購入導線だが、**LP設計思想の参考**になる。

| 要素 | 北の快適工房での例 | 転用の可否 |
|------|-------------------|-----------|
| コンセプト一文 | 「びっくりするほど良い商品ができた時にしか発売しない」 | 思想は借りるが文言は不可 |
| 悩み別導線 | 悩み別・カテゴリ別で探す | 工種・級・intentチップ（当社） |
| ストーリーLP | モノづくりの考え方（長文・共感） | 体験談・`<details>`任意 |
| 定期・個数UI | 「1〜3個」で割引率が変わる表示 | 数量ではなく「10分・30秒」マイクロコピー |
| 返報・不安 | 解約可能・初回のみ必須の明示 | サンクス・予約前チェックリスト |
| 社長直轄 | 北の達人は強い | 当社は**厚労省許可・工種特化**で代替 |

参考: [コンセプト](https://www.kaitekikobo.jp/concept/)、[商品LP例](https://www.kaitekikobo.jp/hb/)

## 6. 木下流 LP構成（一般論＋当社マッピング）

### 6.1 3ブロック（広告LPの定石との接続）

| ブロック | 役割 | 当社 |
|----------|------|------|
| ファーストビュー | 3秒で「自分向け・無料・次」 | FV + `meta-lp` |
| ボディ | 信頼・比較・不安解消 | 口コミ・許可・返報ステップ |
| クロージング | 1アクション | フォーム送信 → **サンクス** |

木下氏: 商品開発と**LPを同時並行**で作ると売れる（D2C）。当社はサンクスをLPの延長として設計。

### 6.2 ロング vs ショート

| チャネル | 木下流の考え | 当社 |
|----------|-------------|------|
| 広告クリック直後 | **ショートが先に勝ちやすい**（3秒） | `meta-lp/` |
| 検討時間あり | ロングも有効 | 通常LP + FAQ下部 |
| 登録直後 | 次の1アクションを1本 | thanks-v2: 予約 → LINE |

### 6.3 サンクスページ（木下流 × 当社 thanks-v2）

| 木下の原則 | thanks-v2での実装 |
|-----------|------------------|
| 絶対伝えたいことは最初 | 登録完了 → 3ステップ → 予約CTA |
| 不安をマイクロコピーで潰す | `t-hero__objections`、予約前 `t-cal__micro` |
| 返報（次に何が得られるか） | 求人概要 → 電話 → 全文 |
| 社会的証明 | ★4.8 strip・体験談（資格別） |
| 1アクションに絞る | 主: 予約 / 副: LINE（ゲート） |
| 4段階の大分類 | 押し売り・面倒をヒーローで潰す |
| テクニカル | GTM: 予約・LINE・工種別 `_lp` |

## 7. チェックリスト（木下流・新規LP / サンクス改修前）

### ファンダメンタルズ

- [ ] 誰向けかが3秒でわかるか（工種・級）
- [ ] 何が無料かが一言でわかるか
- [ ] 広告1行目とFV1行目が一致しているか
- [ ] 「買わない・登録しない理由」を大分類で潰しているか
- [ ] 転職しなくてもOKが伝わるか
- [ ] 絶対伝えたいことはページ上部にあるか

### テクニカル

- [ ] ボトルネックステップだけ直す（レイヤー固定）
- [ ] マイクロコピー（ボタン・ゲート・注釈）を疑い直したか
- [ ] 採算の合わない広告/LPバリアントを止めているか
- [ ] 数値は根拠確認済みか（`CLAUDE.md`）
