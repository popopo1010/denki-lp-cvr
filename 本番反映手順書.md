## Claude Codeで作業する場合の進め方

`Mastering Claude Code in 30 minutes` の要点を、本LPの本番反映作業では以下の運用に落とし込む。

### 1. まずコードベースQ&Aから始める
いきなりHTML/CSS/JSを編集せず、Claude Codeに以下を確認させる。

- 対象ページで読み込まれているHTML/CSS/JSの構造
- 既存フォームのステップ遷移、バリデーション、送信処理
- GTM注入とテーマ反映のどちらが今回の目的に合うか
- 過去の変更理由が不明な箇所は、必要に応じてGit履歴も確認

### 2. 編集前に短い計画を出す
本番影響がある変更では、Claude Codeに「編集前に計画を出して確認を待つ」よう指示する。

- 変更目的
- 変更対象ファイル
- 影響範囲
- 確認方法
- ロールバック方法

### 3. 検証できる単位で反復する
Claude Codeに作業させる場合も、確認手段を先に決めてから進める。

- HTML変更後: 表示崩れ、CTA、フォームステップ遷移を確認
- CSS変更後: SP/PC表示、固定CTA、モーダル、FAQ開閉を確認
- JS変更後: 入力補助、離脱防止、カウントアップ、送信前同意文言を確認
- 数値・許可番号・口コミ・会社情報を変更した場合: 社内確認項目に追加

### 4. 共有コンテキストを更新する
LP固有の判断ルールや本番反映で毎回使う手順は、必要に応じて `CLAUDE.md` または本手順書へ追記する。個人の好みや一時的な作業メモは共有ルールに入れない。

## 3パターンLPの変更ルール

このリポジトリには、通常LP、`WPLP`、`自前LP` の3パターンがある。どれか1つを変更する場合も、他パターンへの影響確認を必ず行う。

| パターン | 主な用途 | CSS/JS |
|---|---|---|
| 通常LP | GitHub Pages / 本番反映確認用 | `assets/css/cvr-boost.css` / `assets/js/app.js` |
| `WPLP` | WordPressテーマCSS併用版 | `WPLP/assets/css/cvr-boost.css` / `WPLP/assets/js/app.js` |
| `自前LP` | WordPress依存なしの単体版 | `自前LP/assets/css/style.css` / `自前LP/assets/js/app.js` |

変更時の原則:
- HTMLだけを変えた場合も、同じLP種別（`denkikouji` / `sekoukanri` / `thanks` / `privacypolicy`）が他パターンにあるか確認する。
- JSのセレクタ、フォームID、classを変える場合は、3パターンすべてのフォームLPで同じ要素が存在するか確認する。
- `自前LP` では外部WordPressテーマCSSに依存させず、原則 `自前LP/assets/css/style.css` を使う。
- 変更後は以下を実行し、相対リンク切れとフォーム構造の破損がないことを確認する。

```bash
node scripts/check-lp-guard.js
```

## 反映方法（2通り）

### 方法A：テーマにCSS/JSファイルを追加（推奨）

#### Step 1: ファイルをサーバーにアップロード
以下の2ファイルをFTP/SSH等でアップロードしてください。

```
wp-content/themes/original-thema/assets/css/cvr-boost.css
wp-content/themes/original-thema/assets/js/app.js
```

ファイルはGitHubリポジトリの以下から取得:
- `assets/css/cvr-boost.css`
- `assets/js/app.js`

#### Step 2: テーマでCSS/JSを読み込む
`functions.php` または該当ページテンプレートの `</head>` 前に追加:

```html
<link rel="stylesheet" href="/wp-content/themes/original-thema/assets/css/cvr-boost.css">
```

`</body>` 前に追加:
```html
<script src="/wp-content/themes/original-thema/assets/js/app.js" defer></script>
```

#### Step 3: 既存の読み込みを削除（軽量化）
以下の外部JSは `app.js` に統合済みのため不要:
- `jquery.min.js` （未使用）
- `gsap.min.js` （CSS transitionに置換済み）
- `main.js` （app.jsに統合済み）

#### Step 4: HTMLの変更
GitHubリポジトリの `denkikouji/index.html` を参考に、
WordPress管理画面で該当ページのHTMLを更新してください。

主な変更箇所:
- フォーム前に「信頼性バー」「社会的証明」「リアルタイム通知」を追加
- 各ステップにマイクロコピーを追加
- フォーム後に「利用者の声」「よくある質問」を追加
- 最終ステップにPP同意テキストを追加
- フローティングCTAを追加
- フッターの許可番号を変更

---

### 方法B：GTMで注入（テーマを触らずに反映）

GTM（GTM-KV525PZ）のカスタムHTMLタグで `cvr-boost.css` と `app.js` を注入。
テーマファイルを一切触らずに反映可能。ロールバックも即座にできる。

#### 手順
1. GTM管理画面にログイン
2. 新しいタグを作成 → カスタムHTML
3. 以下を貼り付け:

```html
<link rel="stylesheet" href="https://popopo1010.github.io/denki-lp-cvr/assets/css/cvr-boost.css">
<script src="https://popopo1010.github.io/denki-lp-cvr/assets/js/app.js" defer></script>
```

4. トリガー: ページURL が `/denkikouji-kyujin-2/` を含む
5. 公開

※GitHub PagesのURLから直接読み込むため、サーバーアップロード不要

## Claude Code作業後の最終チェック

- `git diff` で意図しないファイル変更がないか確認
- 表示確認URLと本番対象URLを取り違えていないか確認
- GTM反映の場合、対象URLトリガーが `/denkikouji-kyujin-2/` に限定されているか確認
- テーマ反映の場合、アップロード先と読み込みパスが一致しているか確認
- ロールバック手順（GTMタグ停止、または追加CSS/JS読み込み削除）を関係者に共有
