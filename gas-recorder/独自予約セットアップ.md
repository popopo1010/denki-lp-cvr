# 独自予約システム（TimeRex不要）

LPの電話・名前を使い、**サンクスで日時だけ選ぶ** → **Googleカレンダーに予定作成** → **スプシ更新 + Slack通知**。

## 構成

```
サンクス（独自UI・3日表示）
  ↓ JSONP 空き枠取得
GAS + Googleカレンダー（担当者の空き）
  ↓ 予約 POST
Googleカレンダーにイベント作成 + スプシ + Slack
```

## 2. 受付ルール（GASで自動）

| 項目 | 既定値 | プロパティで変更 |
|------|--------|------------------|
| 枠の長さ | 15分 | `BOOKING_SLOT_MINUTES` |
| 受付時間 | 平日 9:00〜24:00（23:45枠まで） | `BOOKING_START_HOUR` / `BOOKING_END_HOUR`（既定 9 / 24） |
| 重複予約 | 既定OFF（空いてる担当のみ枠表示・予約） | `BOOKING_ALLOW_OVERLAP=true` で被りOKに戻せる |
| 担当割当 | その枠で空いている人のうち **当日予定が少ない人優先**（同数はRR） | 自動 |
| 何時間後から | 2時間後 | `BOOKING_LEAD_HOURS` |
| 何日先まで | 14日（サンクスAPIは5日） | `BOOKING_DAYS_AHEAD` |

空き判定は **そのカレンダーの既存予定と重ならない枠** のみ表示します。

### 表示速度（Cache + 静的JSON）

| 層 | 内容 |
|----|------|
| GAS `CacheService` | 空き枠レスポンスを **3分**キャッシュ（`booking-custom.js`） |
| 時間トリガー | `warmBookingSlotsCache` を **5分ごと**（推奨） |
| 静的JSON | `assets/data/booking-slots.json` … deploy / 5分CIで更新 |
| ブラウザ | 静的JSON → 失敗時GAS JSONP、sessionStorage 5分 |

## 5. 担当者複数（空きマージ＋ラウンドロビン）— 方式A

求職者には **日時だけ** 表示。裏で柴山・林・福山・山田の **カレンダーが空いている枠だけ** をマージ表示し、予約確定時は **空いている担当のうち当日の予定が一番少ない人** に入れます（同数ならRR）。

### スクリプトプロパティ `BOOKING_STAFF_JSON`

GAS → プロジェクトの設定 → スクリプトプロパティ → 新規:

| 名前 | 値（1行JSON・改行なし） |
|------|-------------------------|
| `BOOKING_STAFF_JSON` | 下記例をコピーし、各 `calendar_id` を実IDに差し替え |

```json
[{"id":"shibayama","name":"柴山","calendar_id":"yuki.shibayama@xchange-inc.com"},{"id":"hayashi","name":"林","calendar_id":"林のカレンダーID"},{"id":"fukuyama","name":"福山","calendar_id":"福山のカレンダーID"},{"id":"yamada","name":"山田","calendar_id":"山田のカレンダーID"}]
```

- `calendar_id` が空・無効な担当は **自動でスキップ**（柴山だけでも動作）
- 未設定のときは従来どおり `BOOKING_CALENDAR_ID` / デフォルトカレンダー1本

### カレンダーIDの調べ方

1. 各担当の Googleカレンダーを **GAS実行アカウント**（`yuki.shibayama@xchange-inc.com`）と共有（変更権限）
2. GASエディタで **`getBookingStaffInfo`** を実行 → ログに `staff[]` が出る
3. 反映後: **`clasp push`** → **再デプロイ**（既存デプロイID）→ **`warmBookingSlotsCache`** トリガー継続

### スプシ・Slack

- 列追加: `calendar_staff_id`, `calendar_staff_name`
- Slack: `*担当:* 林` を付与
- カレンダー件名: `【LP面談・林】山田 太郎 090...`

### 負荷

- ユーザー: JSON 1本のまま（枠はマージされるので **空きは増えやすい**）
- GAS: 担当数ぶん `getEvents`（5分キャッシュで平準化）
